1. Typography Hierarchy (The Text)

Good UX depends on a clear Visual Hierarchy. You want the user's eyes to know exactly what to read first.

    H1 (Page Title): The largest text. There should usually only be one per page so the user knows the main topic immediately.

    H2 & H3 (Subtitles): These break up the page into sections. They help users "scan" the page without reading every word.

    Body Text (Paragraph): The main content. It needs to be a readable size (usually 16px or larger) with enough "Line Height" (space between lines) so it doesn't look cramped.

    Captions/Labels: Very small text used for image descriptions or hints inside input boxes.

2. Login & Security Elements

The login flow is one of the most sensitive parts of UX. If it's frustrating, users will leave.

    Input Fields: Boxes for the Username/Email and Password. They should have "Placeholder" text (faint text inside the box) to show what to type.

    Show/Hide Password: That little "eye" icon. It’s essential for mobile users who might mistype their password.

    "Forgot Password" Link: Always placed near the login button. It’s the most clicked link when a user is frustrated!

    Social Login Buttons: "Sign in with Google/Apple" buttons. These reduce "friction" by letting users skip the long sign-up form.

    Validation States: The red text that appears if you forget the "@" in an email or if the password is too short.

3. List & Organization Elements

Lists help you display a lot of information in an orderly way.

    Unordered Lists (Bulleted): Used when the order doesn't matter (like a list of features).

    Ordered Lists (Numbered): Used for step-by-step instructions or rankings.

    Data Tables: For comparing complex info (like pricing plans or flight times). These need clear column headers.

    Dropdown Lists (Select Menus): Used when you have many options (like choosing a Country) but want to save space on the screen.

    Checklists: Often used in "To-Do" apps or onboarding flows to show progress.

4. Spacing & Grid (The Framework)

    The Grid System: Designers use an invisible grid (usually 12 columns) to make sure everything aligns perfectly.

    Padding & Margins: This is the "white space" around your text and buttons. Without it, your site looks like a messy wall of text.

    Pro Tip: In UI design, Consistency is King. If your "H2" subtitle is Blue and Bold on the Home page, it must be Blue and Bold on every other page too.



1. Navigation Components (How users move around)

    Navbar (Header): The menu at the top. It should include your logo (linked to home) and links to the main pages.

    Footer: The section at the bottom for legal links, contact info, social media, and a sitemap.

    Search Bar: Essential for sites with lots of content so users can find things quickly.

    Breadcrumbs: Small links (e.g., Home > Shop > Shoes) that show the user exactly where they are in the site's hierarchy.

    Hamburger Menu: The "three-line" icon used on mobile to hide the menu and save space.

2. Input & Action Components (How users interact)

    Buttons: You need "Primary" buttons (for the main action like "Buy Now") and "Secondary" buttons (for less important actions like "Cancel").

    Forms & Input Fields: For logins, contact pages, or checkouts. These must have clear labels and error messages.

    Call to Action (CTA): A prominent button or link designed to get the user to do one specific thing (e.g., "Sign Up Free").

    Checkboxes & Radio Buttons: For selecting options (Checkboxes = select multiple; Radio buttons = select only one).

3. Content & Information Components (How data is shown)

    Cards: Rectangular containers used to group related information together (like a product image, title, and price).

    Modals (Pop-ups): Overlays that appear for important tasks, like a login window or a confirmation message.

    Accordions: Sections that expand and collapse to hide/show details (common in FAQ sections).

    Icons: Visual symbols (like a magnifying glass for search) that help users understand actions without reading text.

4. Feedback & Status (How the site talks back)

    Loaders/Spinners: Icons that show the user the site is working so they don't think it's broken.

    Toast Notifications/Alerts: Small pop-up messages that say "Success!" or "Error: Try again."

    Empty States: What the user sees when there is no data (e.g., "Your cart is empty").

5. Essential UX Principles (The "Invisible" Needs)

    Responsive Design: The site must look good and work perfectly on phones, tablets, and desktops.

    Visual Hierarchy: Using different font sizes and colors so the user knows what to look at first.

    Accessibility (a11y): Ensuring people with disabilities (like color blindness) can use the site. This includes high color contrast and "Alt text" for images.

    White Space: Leaving enough "breathing room" between elements so the site doesn't feel cluttered or confusing.
