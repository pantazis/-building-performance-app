OpenBEP4EU Journey Map

This diagram is the schema/flow overview used to organize all screens.

100%

View Index (Logical Order)

1) Login

What this view is: Entry screen for authentication before using the application.

What you can do: Sign in and proceed to the main project workspace.

Relationship to schema: Practical system entry point before flow node A (Projects Dashboard).

Open view/login.html

2) Projects Dashboard (A)

What this view is: Main control center listing building projects and scores/statuses.

What you can do: Start all major actions (add, open/edit, duplicate, delete, library access).

Relationship to schema: Root node A in the plan, branching to B/C/D/E/F/G/H.

Open view/A-projects-dashboard.html

3) Add New Building (B)

What this view is: Creation form for first-time building setup.

What you can do: Provide basic building metadata and create a new record.

Relationship to schema: Node B; after creation, workflow hands off to Building Tab (I).

Open view/B-add-new-building.html

4) Open / Edit Building (C)

What this view is: Building editing screen for an existing project.

What you can do: Update building profile fields and prepare for deeper EPB/SRI work.

Relationship to schema: Node C that also transitions into Building Tab (I).

Open view/C-open-edit-building.html

5) Global Resource Library (F)

What this view is: Shared catalog for reusable assets and references.

What you can do: Manage reusable materials/components used across multiple buildings.

Relationship to schema: Node F branch from dashboard A supporting reuse-first workflows.

Open view/F-global-resource-library.html

6) Building Tab (I)

What this view is: Main per-building tab hub where EPB and SRI paths begin.

What you can do: Select EPB or SRI dashboard and continue technical assessments.

Relationship to schema: Node I, receiving flow from B/C and routing to J (EPB) and K (SRI).

Open view/I-building-tab.html

7) EPB View 1: Calculation Settings (EPB1)

What this view is: Initial EPB simulation configuration screen.

What you can do: Define time period, simulation resolution, and core calculation settings.

Relationship to schema: EPB sequence start: J → EPB1 → EPB2 → EPB3 → EPB4.

Open view/EPB1-calculation-settings.html

8) EPB View 2: Materials & Constructions (EPB2)

What this view is: Envelope/material definitions for the energy model.

What you can do: Configure walls, windows, layers, and construction assemblies.

Relationship to schema: Second EPB step after settings; feeds model physics inputs.

Open view/EPB2-materials-constructions.html

9) EPB View 3: Thermal Zones & Envelope Analysis (EPB3)

What this view is: Thermal zoning and operational definition screen.

What you can do: Set zones/spaces, occupancy/equipment assumptions, and envelope analysis inputs.

Relationship to schema: Third EPB stage immediately before results summary (EPB4).

Open view/EPB3-thermal-zones-envelope.html

10) EPB Results / Performance Summary (EPB4)

What this view is: EPB outcome view with final energy-performance outputs.

What you can do: Review calculated indicators and summary KPI results.

Relationship to schema: EPB chain endpoint and recipient of engine output (CE6 → EPB4).

Open view/EPB4-results-summary.html

11) SRI View 1: Methodology Selection (SRI1)

What this view is: Entry point for Smart Readiness workflow configuration.

What you can do: Choose methodology path and initialize the SRI assessment process.

Relationship to schema: SRI chain start after Building Tab, before weighting decision flow.

Open view/SRI1-methodology-selection.html

12) SRI Decision: User-defined Weightings (SRI2)

What this view is: Decision gate for using default or custom impact weightings.

What you can do: Select whether to proceed with user-defined weighting strategy.

Relationship to schema: Transition node between methodology selection and detailed weighting setup.

Open view/SRI2-user-defined-weightings.html

13) SRI View 2: Weighting Settings (SRI3)

What this view is: Configuration interface for SRI impact/category weighting values.

What you can do: Adjust and confirm custom weighting parameters used in scoring.

Relationship to schema: Follows SRI2 custom path and feeds the domain dashboard.

Open view/SRI3-weighting-settings.html

14) SRI View 3: Domain Dashboard (SRI4)

What this view is: Central SRI domain navigator across technical service tabs.

What you can do: Review readiness progress and jump into each service domain form.

Relationship to schema: SRI hub branching to service tabs SRI5 through SRI13.

Open view/SRI4-domain-dashboard.html

15) Heating Tab (SRI5)

What this view is: Heating services smart-readiness assessment tab.

What you can do: Enter functionality levels and controls for heating-related criteria.

Relationship to schema: One of the service-domain children of SRI4.

Open view/SRI5-heating-tab.html

16) DHW Tab (SRI6)

What this view is: Domestic hot water smart-readiness assessment tab.

What you can do: Configure DHW service functionality and monitoring/control levels.

Relationship to schema: Service-domain branch from SRI4 in the SRI workflow set.

Open view/SRI6-dhw-tab.html

17) Cooling Tab (SRI7)

What this view is: Cooling systems smart-readiness assessment tab.

What you can do: Set cooling automation, efficiency, and control capability levels.

Relationship to schema: Service-domain branch under the SRI4 dashboard.

Open view/SRI7-cooling-tab.html

18) Ventilation Tab (SRI8)

What this view is: Ventilation services smart-readiness input tab.

What you can do: Record ventilation capabilities for scoring and domain completeness.

Relationship to schema: Another technical-domain tab routed from SRI4.

Open view/SRI8-ventilation-tab.html

19) Lighting Tab (SRI9)

What this view is: Lighting services smart-readiness tab.

What you can do: Capture lighting control/intelligence functions for SRI computation.

Relationship to schema: Domain-specific node in the SRI4 service-tab group.

Open view/SRI9-lighting-tab.html

20) Dynamic Envelope Tab (SRI10)

What this view is: Assessment tab for adaptive envelope/building shell functions.

What you can do: Define dynamic envelope capabilities contributing to SRI score.

Relationship to schema: Domain branch from SRI4 alongside other service tabs.

Open view/SRI10-dynamic-envelope-tab.html

21) Electricity Tab (SRI11)

What this view is: Electricity-domain smart-readiness data-entry tab.

What you can do: Configure electrical service intelligence and responsiveness levels.

Relationship to schema: SRI4 child node for electricity-related functionalities.

Open view/SRI11-electricity-tab.html

22) EV Charging Tab (SRI12)

What this view is: Electric-vehicle charging readiness assessment tab.

What you can do: Enter EV charging intelligence and interaction features.

Relationship to schema: Service-domain branch in the SRI tabbed sequence.

Open view/SRI12-ev-charging-tab.html

23) Monitoring & Control Tab (SRI13)

What this view is: Monitoring, automation, and control capability assessment tab.

What you can do: Define supervisory/control features that influence smart readiness.

Relationship to schema: Final service-domain input tab before SRI results aggregation.

Open view/SRI13-monitoring-control-tab.html

24) SRI Results (SRI14)

What this view is: Final SRI output screen with consolidated smart-readiness indicators.

What you can do: Review total score, domain performance, and final readiness class.

Relationship to schema: Endpoint of SRI workflow after domain/service evaluation.

Open view/SRI14-results.html

25) About

What this view is: Informational/support page describing the app context.

What you can do: Read project information and orientation content.

Relationship to schema: Auxiliary page, outside the core A→EPB/SRI transactional path.

Open view/about.html
